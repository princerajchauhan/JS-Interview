Q1. Difference between “ == “ and “ === “ operators.

  • The == operator does the type conversion of the operands before comparison.
  • the === operator compares the values as well as the data types of the operands.
Example :-
            
            let a = 5
            let b = '5'

            console.log(a == b)    // true
            console.log(a === b)   // false

Q2. What are the differences between var, let and const?

  • var and let create variables that can be reassigned another value. const creates "constant" variables that cannot be reassigned another value
Example :-
            
            {
                var a =10;
                let b =20;
                const c =30;
            }
            console.log(a);  //10
            console.log(b);  // b is not defined
            console.log(c);  // c is not defined
            
            let d = 10;
            let d = 20; // SyntaxError: Identifier 'd' has already been declared
            d = 11;
            console.log(d); // 11
                
            const e = 10;
            e = 11;
            console.log(e); // Assignment to constant variable

  • The reason that we get a reference error when we try to access a let or const variable before its declaration is because of the temporal dead zone (TDZ).

Q2. What is an event loop and call stack

  • Event Loop: Then comes the Event Loop, which keeps running continuously and checks the Main stack, if it has any frames to execute, if not then it checks Callback queue, if Callback queue has codes to execute then it pops the message from it to the Main Stack for the execution.
  • Call Stack: This is where all your javascript code gets pushed and executed one by one as the interpreter reads your program, and gets popped out once the execution is done.
Example :-
            
            console.log('Message 1');   // Execute 1st
            // Print message after 100 millisecond
            setTimeout(function() {
                console.log('Message 2');       
            }, 100);
            console.log('Message 3');   // Execute 2nd

Q4. What is the spread operator?

  • Spread operator allows us to destructure the non-primitive datatype like object and array to access elemnets individually.
  • The JavaScript spread operator ( ... ) allows us to quickly copy all or part of an existing array or object into another array or object.
Example :-
            
            let a = [2, 5, 6]

            let c = [...a]  // spred operator - deep copy

Q6. What are callbacks?

  • A callback is a function that passed as an argument to another function.

Q7. What is callback hell

  • Callback Hell is essentially nested callbacks stacked below one another forming a pyramid structure.
Example :-
            
            setTimeout(()=>{
                console.log("A")
                setTimeout(()=>{
                    console.log("B")
                    setTimeout(()=>{
                        console.log("C")
                        setTimeout(()=>{
                            console.log("D")
                            setTimeout(()=>{
                                console.log("E")
                            },5000)
                        },4000)
                    },3000)
                },2000)
            },1000)

Q1. What are promises and why do we need them?

  • Promise is an object that returns a value which you to receive in future.
  • Promise are use to handle async operation in JS. easy to handle callback hell, also use to handle the error.
  • basically in promise there are Three stage
    • Pending - initial stage
    • Resolve - (then method)
    • Reject - (catch method)
Example :-
            
            const isFine = false
            const promise = new Promise((resolve, reject) => {
                if(isFine){
                    resolve("Everything is fine")
                }
                else{
                    reject("Something error")
                }
            })
            
            promise.then(data => console.log(data))
            .catch(err => console.log(err))

Q2. What is promise chaining

  • Promise chaining is a technique in JavaScript to execute multiple asynchronous operations in a specific order by chaining promises together using the then method
Example :-
            
            function promiseFun(text, timeOut){
                return new Promise((resolve, reject) => {
                    setTimeout(() => {
                        console.log(text)
                        resolve()
                    }, timeOut)
                })
            }
                        
            promiseFun("Prince", 1000)
            .then(() => promiseFun("Raj", 2000))
            .then(() => promiseFun("Chauhan", 3000))
            .then(() => promiseFun("Hello", 2000))
